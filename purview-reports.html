Key Takeaway

"Accessed file" in Purview reports does not mean someone opened or downloaded the file. It often indicates system-level activities like background scans, indexing, or automated processes. Understanding the difference between system access and user actions is crucial for accurate interpretation.

‚úÖ What "Accessed file" Actually Means

When you see "Accessed file" entries in Microsoft Purview reports, it's important to understand what this activity represents:

  • System searches and indexing
    (Background processes that scan files for search functionality, malware detection, or content analysis.)
  • Preview generation
    (Automatic creation of file previews for SharePoint, Teams, or OneDrive interfaces.)
  • Compliance scanning
    (Automated checks for data classification, retention policies, or regulatory compliance.)
  • Backup and synchronization
    (System processes that ensure data is properly backed up and synchronized across services.)
  • Security monitoring
    (Automated security scans and threat detection processes.)
Important: These are typically automated system activities, not user-initiated actions like opening or downloading files.

‚ùå What "Accessed file" Does NOT Mean

It's crucial to understand what "Accessed file" entries do not indicate:

  • Someone opened the file
    (The user did not necessarily view, read, or interact with the file content.)
  • File was downloaded
    (The file was not necessarily saved to a user's device or local storage.)
  • User-initiated action
    (The activity was likely automated by Microsoft services, not triggered by a person.)
  • Data breach or unauthorized access
    (These are normal system operations that occur as part of Microsoft 365's functionality.)
  • Privacy violation
    (System access is part of the service you're using and is covered by Microsoft's privacy policies.)
Important: Don't assume "Accessed file" means human interaction with the file content.

Understanding IP Addresses in Purview Reports

IP addresses in Purview reports can help you distinguish between user activity and system activity.

Microsoft Service IPs: These are IP addresses owned by Microsoft that indicate automated system processes.

Think of it like the difference between a delivery truck (Microsoft service) visiting your building versus a person (user) entering your office. Both are recorded, but they serve very different purposes.

Microsoft Service IPs

  • Indicate automated system processes
  • Often appear in ranges (e.g., 13.107.x.x)
  • Associated with background scans and indexing
  • Normal part of Microsoft 365 operation
  • Not user-initiated activity

User IP Addresses

  • Show actual user activity
  • May be from office, home, or mobile networks
  • Associated with direct file interactions
  • Indicate human-initiated actions
  • Can be traced to specific users

Frequently Asked Questions

Common questions about interpreting Microsoft Purview reports and "Accessed file" entries.

Report Interpretation

  • Q: Did the user open the file?
    A: Not necessarily. "Accessed file" often means system-level access for indexing, scanning, or preview generation.
  • Q: Why are there only 10 rows when I saw more files?
    A: Purview reports may be filtered or limited. Check your date range, user filters, and activity types.
  • Q: What does SharePointFileOperation mean?
    A: This indicates a SharePoint-specific operation, often automated system processes like indexing or compliance scanning.
  • Q: Is this a GDPR problem?
    A: Generally no. System access is covered by data processing agreements and serves legitimate business purposes.

Technical Details

  • Q: How can I tell if it's system vs user access?
    A: Look for "system@" users, Microsoft IP addresses, and automated client applications.
  • Q: Why do I see multiple entries for the same file?
    A: Different system processes may access the same file for various purposes (indexing, scanning, previews).
  • Q: Can I filter out system activity?
    A: Yes, you can filter by user (exclude system@ accounts) or by specific activity types.
  • Q: How long are these logs kept?
    A: Audit logs are typically retained for 90 days to 1 year, depending on your Microsoft 365 license.

Key Points to Remember

System vs User: Always distinguish between automated system processes and actual user actions when interpreting audit logs.

Context Matters: Consider the IP address, user account, and client application to understand the nature of the activity.

Normal Operation: System access is a normal part of Microsoft 365 functionality and doesn't indicate unauthorized access.

Glossary of Purview Terms

Definitions of key terms used in Microsoft Purview reports

Activity

The specific action that was performed, such as "Accessed file," "Downloaded file," or "Modified file." The type of activity helps determine if it was user-initiated or system-generated.

Operation

The technical operation performed by the system, such as "SharePointFileOperation" or "ExchangeItemOperation." Operations provide more detail about the specific system process.

ClientIP

The IP address from which the activity originated. Microsoft service IPs indicate automated processes, while user IPs show actual user activity.

Accessed File

An audit entry indicating that a file was accessed by the system or a user. This does not necessarily mean the file was opened or downloaded by a person.

System Account

An automated account (often starting with "system@") that performs background processes like indexing, scanning, or compliance checks. These are not human users.
